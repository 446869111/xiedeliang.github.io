Android内存检测工具MAT的使用

发表于 2016-03-09

前言

  • 内存的检测与分析的方法和工具有很多，比如Lint、PMD、Emmagee、DDMS、Android Monitor、手机中开发者选项的设置等等，在这里只简单介绍一下MAT(Memory Analysis Tools)的使用。当然了其他的工具后期也会介绍。

    正文

  • DDMS 可以将当前的内存 Dump成一个hprof格式的文件，MAT 读取这个文件后会给出方便阅读的信息，配合它的查找，对比功能，就可以定位内存泄漏的原因。

  • MAT下载地址：https://www.eclipse.org/mat/

1.hprof文件的获取

  • 打开Android Device Monitor,然后按图示选择


  • 转换 hprof文件

    DDMS Dump 出的文件要经过转换才能被 MAT识别，Android SDK提供了这个工具 hprof-conv (位于 sdk/tools下，eclipse导出的无需转换)

    打开命令提示符（CMD）切换到tools目录下，执行hprof-conv aaa.hprof bbb.hprof即可完成转换，aaa.hprof 代表转换前的原文件，bbb.hprof代表转换后的。

    到此MAT工具的使用就简单的介绍完了，当然了，重点是分析.hprof文件，这个在下一节介绍。

总结：

  • MAT可通过分析.hprof文件来确定内存泄漏情况
  • MAT下载地址：https://www.eclipse.org/mat/
  • 通过Android Device Monitor获取.hprof文件并用sdk/tools下的hprof-conv.exe工具进行转换
  • 通过MAT来分析.hprof文件查找内存泄漏情况

List对象内容的比较

发表于 2016-02-26

###1.我们在开发的过程中总会遇到两个对象的比较，或者两个数组的比较，对象还好说，比较起来比较简单，但数组呢？

###大家都知道对象的比较有两种方式:”==”和”equals”,那么他们之间有什么区别么？在数组中又应该注意什么呢？

####1.”==”
Java中的”==”是用来比较两个对象的地址是否一样，与普通的基本类型不一样，在基本类型中（如int）比较的是对应的值,在封装类中使用场景不多。

####2.”equals”
Java中的”equals”是比较两个对象的内容是否相同，由于所有的类都是继承java.lang.Object类的，所以如果没有对该方法进行覆盖的话，调用的仍然是Object类中的方法。而Object中的equal方法返回的却是==的判断，因此，如果在没有进行该方法的覆盖后，调用该方法是没有任何意义的。在java面向对象的处理中我们一般在javaBean中都要选择重写equals方法，比较如下：

  • 基本数据类型“==”比较的是内容，封装类比较的是地址。
  • equals 比较的是两个对象的内容，但需要重写其equals和hasCode方法，否则比较的是基类中“==”的比较结果。

Hexo 常用命令的使用

发表于 2016-02-22

1.安装

* 利用 npm 命令即可安装。（在任意位置点击鼠标右键，选择Git bash）
* npm install -g hexo

2.创建hexo文件夹

* 安装完成后，在你喜爱的文件夹下（如H:\hexo），执行以下指令(在H:\hexo内点击鼠标右键，选择Git bash)，Hexo 即会自动在目标文件夹建立网站所需要的所有文件。
* hexo init

3.安装依赖包

* npm install

4.本地查看

* 现在我们已经搭建起本地的hexo博客了，执行以下命令(在H:\hexo)，然后到浏览器输入localhost:4000看看
* hexo generate
* hexo server

5.保存修改的文件及部署到gitHub上

* hexo generate
* hexo deploy

6.新建博客

* hexo new "AAAA"
* 在hexo\source\_posts中打开文件
* hexo server，访问localhost:4000预览效果。（退出server用Ctrl+c）
* hexo deploy，同步到github。访问网站看看效果。

7.异常处理

* npm install hexo-deployer-git --save
