Linux下Android环境搭建

发表于 2018-02-10 | 分类于 环境

最近在研究linux下搭建Android编译环境，用来自动化发版打包，由于自己对命令行不是很敏感，所以还是走了一些弯路，在这里记录一下。

由于是搭建编译环境用来打包，采用的是在centos下虚拟机

由于这里只是用来编译，并不涉及到开发，所以只安装了能够编译的环境，具体如下：

  • JDK
  • SDK
  • Gradle

安装前准备

  • 1、 Android sdk-linux
  • 2、 gradle
  • 3、 jdk

首先安装jdk

  1. 利用yum源来安装jdk（此方法不需要配置环境变量）

    查看yum库中的java安装包 ：yum -y list java*

    安装需要的jdk版本的所有java程序：yum -y install java-1.8.0-openjdk*


    安装完成，可以使用java -version 查看当前jdk版本

    卸载使用：

    yum -y remove java-1.8.0-openjdk*

  1. 使用Xftp将下载好的jdk-8u191-linux-x64.rpm 上传到CentOs上，然后使用rpm安装

    Xftp安装指引参考链接

    我这里放到了/home/xdl文件夹下，查看一下文件

    使用rpm命令安装： rpm -ivh jdk-8u191-linux-x64.rpm

    卸载使用：

    rpm -qa | grep jdk 查询已安装jdk名称
rpm -e --nodeps jdk1.8 来卸载

  2. 使用Xftp将下载好的jdk-8u151-linux-x64.tar.gz上传到CentOs上，然后使用解压以及配置环境变量的方式进行安装

    1. 复制安装包到/usr/develop/java目录中(目录可以自己选)：
      cp jdk-8u151-linux-x64.tar.gz /usr/develop/java
    2. 解压缩包：tar -zxvf jdk-8u151-linux-x64.tar.gz

      解压后jdk对应的文件夹为jdk1.8.0_151

    3. 配置环境变量

      输入：vim /etc/profile
      向文件里面追加以下内容：

      复制如下内容
      #set java environment JAVA_HOME=/usr/develop/java/jdk1.8.0_151 JRE_HOME=$JAVA_HOME/jre PATH=$PATH:$JAVA_HOME/bin:$JRE_HOME/bin CLASSPATH=.:$JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib/tools.jar:$JRE_HOME/lib export JAVA_HOME JRE_HOME PATH CLASSPATH

    4. 使配置文件生效：source /etc/profile

安装 Gradle

将下载下来的gradle-4.1-all.zip文件使用ftp上传到云主机上，我这里放在了/home/xdl文件夹下

解压gradle-4.1-all.zip(为了整齐，我把所有开发工具都放到/server文件夹下面了)

  • 将文件copy到自定义(/server）的文件夹下
  • 使用命令unzip gradle-4.1-all.zip进行解压

解压后对应的gradle-4.1文件夹就是我们要的文件夹，具体如下：

配置gradle环境变量

vim /etc/profile

在文件中增加GRADLE_HOME输出路径,并且修订PATH设置如下

使profile 生效：source /etc/profile

查看gradle配置是否生效：gradle -version

安装 Android SDK

解压SDK到目录/server/androidSDK

配置SDK环境变量

vim /etc/profile

在文件中增加GRADLE_HOME输出路径,并且修订PATH设置如下:

export ANDROID_HOME=/server/androidSDK
export PATH=$ANDROID_HOME/tools:$ANDROID_HOME/platform-tools:$JAVA_HOME/bin:$GRADLE_HOME/bin:$PATH

使profile 生效：source /etc/profile

查看SDK配置是否生效：android

以上就已经全部安装完成了，当然我这个不需要安装Android Studio 我只是需要一个编译环境，如果需要请自行安装
