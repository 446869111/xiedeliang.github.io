CentOS上安装Jenkins

发表于 2018-03-14

买了一个腾讯云的主机，闲来无事，开始折腾CentOS上集成个Jenkins环境。

登录云主机，查看当前主机系统，如果不是CentOS可以选择重新安装系统，我这里重装了一下，改成了CentOS7,稍等一会系统就会安装好了，接下来登录云主机，开始搞起来吧。

我是用的是Mac下的命令行窗口进行的登录，第一次登录遇到了麻烦，会提示以下错误

经过一番了解才知道原来我在重装系统前使用我的Mac连接过之前的云主机，现在换了新装的云主机系统，那么这些系统使用同一ip，登录过一次后就会把ssh信息记录在本地的~/.ssh/known_hsots文件中，切换该系统后再用ssh访问这台主机就会出现冲突警告。

有以下两个解决方案：

  1. 手动删除修改known_hsots里面的内容；
  2. 修改配置文件“~/.ssh/config”，加上这两行，重启服务器。
    StrictHostKeyChecking no
    UserKnownHostsFile /dev/null

优缺点：

  1. 需要每次手动删除文件内容，一些自动化脚本的无法运行（在SSH登陆时失败），但是安全性高；
  2. SSH登陆时会忽略known_hsots的访问，但是安全性低；

好了登录进去了，现在开始安装环境吧

手动安装jdk环境

  1. 利用yum源来安装jdk（此方法不需要配置环境变量）

    查看yum库中的java安装包 ：yum -y list java*

    安装需要的jdk版本的所有java程序：yum -y install java-1.8.0-openjdk*


    安装完成，可以使用java -version 查看当前jdk版本

    卸载使用：

    yum -y remove java-1.8.0-openjdk*

  1. 使用Xftp将下载好的jdk-8u191-linux-x64.rpm 上传到CentOs上，然后使用rpm安装

    Xftp安装指引参考链接

    我这里放到了/home/xdl文件夹下，查看一下文件

    使用rpm命令安装： rpm -ivh jdk-8u191-linux-x64.rpm

    卸载使用：

    rpm -qa | grep jdk 查询已安装jdk名称
rpm -e --nodeps jdk1.8 来卸载

  2. 使用Xftp将下载好的jdk-8u151-linux-x64.tar.gz上传到CentOs上，然后使用解压以及配置环境变量的方式进行安装

    1. 复制安装包到/usr/develop/java目录中(目录可以自己选)：
      cp jdk-8u151-linux-x64.tar.gz /usr/develop/java
    2. 解压缩包：tar -zxvf jdk-8u151-linux-x64.tar.gz

      解压后jdk对应的文件夹为jdk1.8.0_151

    3. 配置环境变量

      输入：vim /etc/profile
      向文件里面追加以下内容：

      复制如下内容
      #set java environment JAVA_HOME=/usr/develop/java/jdk1.8.0_151 JRE_HOME=$JAVA_HOME/jre PATH=$PATH:$JAVA_HOME/bin:$JRE_HOME/bin CLASSPATH=.:$JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib/tools.jar:$JRE_HOME/lib export JAVA_HOME JRE_HOME PATH CLASSPATH

    4. 使配置文件生效：source /etc/profile

安装Jenkins

  1. 添加yum repos，然后安装

    1. sudo wget -O /etc/yum.repos.d/jenkins.repo http://pkg.jenkins-ci.org/redhat/jenkins.repo
    2. sudo rpm --import https://jenkins-ci.org/redhat/jenkins-ci.org.key
    3. sudo yum install jenkins

  2. 配置jenkis的端口

    vi /etc/sysconfig/jenkins

    找到修改端口号：
JENKINS_PORT="8989"  此端口不冲突可以不修改

  3. 启动Jenkins

    service jenkins start/stop/restart

    • 安装成功后Jenkins将作为一个守护进程随系统启动
    • 系统会创建一个“jenkins”用户来允许这个服务，如果改变服务所有者，同时需要修改/var/log/jenkins, /var/lib/jenkins, 和/var/cache/jenkins的所有者
    • 启动的时候将从/etc/sysconfig/jenkins获取配置参数
    • 默认情况下，Jenkins运行在8080端口，在浏览器中直接访问该端进行服务配置
    • Jenkins的RPM仓库配置被加到/etc/yum.repos.d/jenkins.repo

  4. 打开Jenkins

    • 在浏览器中访问:ip:8989
    • 首次进入会要求输入初始密码如下图
    • 初始密码在：/var/lib/jenkins/secrets/initialAdminPassword

  5. 安装插件

    • 选择“Install suggested plugins”安装默认的插件，下面Jenkins就会自己去下载相关的插件进行安装。
